## Optimized Assessment Workflow

1. **Enumerate the environment first.** Gather fingerprints (tech stack, exposed services, auth mechanisms, data sensitivity, user roles, hosting model) using lightweight discovery commands and the MCP server tools before selecting tests.
2. **Map findings to the knowledge base.** Use the observed technologies and threat scenarios to filter or rank relevant test cases. Prioritize those that align with the confirmed attack surface or historically high-risk components.
3. **Plan high-value coverage.** Start with the smallest set of test cases that maximizes coverage of critical assets (auth, data exposure, remote code paths). Note which MCP server endpoints/tools exercise each case.
4. **Execute iteratively.** Run the prioritized cases, record evidence, then reassess gaps. Only expand to lower-priority cases if new findings warrant it or coverage remains insufficient.
5. **Continuously sync knowledge.** When novel techniques or environment-specific nuances emerge, add or update test cases via the knowledge base endpoints so future runs benefit from the optimized path.

## Testing Methodologies

- **Web Application Security Assessment:** Perform reconnaissance (DNS, tech stack, hosting), map the attack surface with crawling and proxy logging, validate authentication/session and input handling, verify business-logic and data-exposure flaws, then document proof-of-concepts with reproducible steps.
- **API Security Assessment:** Enumerate hosts and documentation, brute-force hidden endpoints, validate authentication and authorization flows (including token reuse), fuzz parameters for injection or deserialization issues, stress rate limiting, and record all request/response evidence.
- **Mobile Application Security Assessment:** Conduct static review of packages and configuration, run dynamic testing with instrumented proxies/devices, hook critical functions (auth, crypto, storage) using Frida/Drozer/Needle, assess backend/API trust boundaries, and capture screenshots, logs, and patched artifacts.
- **Windows Pentesting:** Start with network/domain reconnaissance, capture credentials or tokens, enumerate local/AD misconfigurations, attempt privilege escalation and lateral movement while logging command outputs, and provide remediation tied to each finding.
- **Linux Pentesting:** Enumerate services, users, sudoers, cron jobs, and file permissions, test common privilege-escalation vectors (kernel exploits, SUID, capabilities), validate persistence or data-access impact, and deliver mitigation guidance per issue.
- **Network Pentesting:** Discover hosts/subnets, fingerprint services, validate vulnerabilities (SMB relay, routing, wireless, VPN, etc.) with controlled exploitation, test segmentation and resilience under simulated attacks, and summarize risks with attack paths.
- **Binary Exploitation:** Classify the binary, gather symbols/strings, perform static and dynamic analysis (GDB, radare2, instrumentation), craft exploits or proofs of concept, and highlight mitigations (ASLR, DEP, stack canaries) alongside bypass steps.
- **Computer Forensics:** Preserve evidence (disk, memory, logs), create timelines, correlate artifacts (registry, browser, process history), detect anomalies or compromise indicators, and produce repeatable findings with hash-verified evidence.
- **Security Code Review:** Define scope and threat models, map data flows per component, run automated scanners for quick wins, perform manual review focusing on high-risk patterns (auth, crypto, input validation), confirm issues via targeted tests, and provide remediation-ready recommendations.
